## Security Summary

### 1. Transport & Headers
| Feature | Detail |
|---------|--------|
| **Helmet.js** | X-Frame-Options: DENY, X-Content-Type-Options: nosniff, HSTS |
| **Next.js headers** | Referrer-Policy, Permissions-Policy (camera self-only, microphone blocked) |
| **CORS** | Configurable via `CORS_ORIGINS` env var, `credentials: true` |
| **Body size limits** | 10 MB max (JSON + URL-encoded) |
| **Compression** | gzip/brotli enabled |

### 2. Authentication
| Feature | Detail |
|---------|--------|
| **Access token** | JWT, 15-minute expiry, stored in HttpOnly/Secure/SameSite cookie |
| **Refresh token** | 40-byte random hex, 7-day expiry, stored in DB + path-restricted HttpOnly cookie (`/api/auth/refresh`) |
| **Token rotation** | Old refresh token deleted on each refresh (prevents reuse) |
| **Logout** | Revokes ALL refresh tokens for user, clears both cookies |
| **Dual extraction** | Cookie-first (web), `Authorization: Bearer` fallback (mobile) |
| **Password hashing** | bcryptjs, 12 rounds |

### 3. Rate Limiting
| Scope | Limit |
|-------|-------|
| **Global** | 60 requests / 60 seconds per IP |
| **Login** | 5 requests / 60 seconds (brute-force protection) |

### 4. Role-Based Access Control (RBAC)
| Endpoint Group | ADMIN-only | All Authenticated |
|----------------|-----------|-------------------|
| **User management** | register, list users, bulk import | update photo, search |
| **Classes** | create, delete | list, update, student ops |
| **Holidays** | create, update, delete | list, check |
| **Reports** | system-status, audit-logs | attendance summaries, grids |
| **Session config** | — | all (controller-level auth) |

### 5. Input Validation
| DTO | Rules |
|-----|-------|
| **LoginDto** | `@IsEmail()`, `@IsString() @MinLength(6)` |
| **RegisterDto** | `@IsEmail()`, password 6–128 chars, name 1–100 chars, `@IsIn(['ADMIN','TEACHER','STUDENT','PARENT'])` |
| **Global pipe** | `whitelist: true`, `forbidNonWhitelisted: true`, `transform: true` |

### 6. Frontend Security
- **`apiFetch()` wrapper** — all API calls use `credentials: 'include'`, auto-refresh on 401, redirect to `/login` on failure
- **AuthGuard** — verifies auth via `/api/auth/me` cookie call, optional role check
- **Zero localStorage tokens** — no JWT stored in JavaScript-accessible storage
- **Concurrent refresh protection** — queues requests during token refresh

### 7. Database
- RefreshToken model with unique constraint, cascade delete on user removal, indexes on `userId` and `expiresAt`
- Production logging restricted to `['error', 'warn']` only

### 8. Potential Enhancements
- **CSP headers** — no explicit Content-Security-Policy yet
- **CSRF** — relying on SameSite cookies (strict in prod) but no explicit CSRF token
- **Per-user rate limiting** — current rate limiting is IP-based only
- **Audit logging** — endpoint exists but not wired to all sensitive operations
- **Secrets management** — SendGrid/Twilio keys should use a vault in production
